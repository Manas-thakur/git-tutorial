## Installing Git

Before using this tutorial, you need Git installed on your system. This guide covers installation on all major operating systems.

### 1. Linux

Git is available via your distribution's package manager.

**Debian / Ubuntu / Linux Mint:**
```
sudo apt update
sudo apt install git
```

**Fedora / RHEL / CentOS:**
```
sudo dnf install git
```

**Arch Linux / Manjaro:**
```
sudo pacman -S git
```

**openSUSE:**
```
sudo zypper install git
```

**Alpine Linux:**
```
apk add git
```

After installation, verify:
```
git --version
```

### 2. macOS

**Option A — Xcode Command Line Tools (simplest):**
```
xcode-select --install
```
A dialog will appear. Click "Install" and wait for it to complete.

**Option B — Homebrew:**
```
brew install git
```

**Option C — MacPorts:**
```
sudo port install git
```

Verify:
```
git --version
```

### 3. Windows

**Option A — Git for Windows (recommended):**
Download from https://git-scm.com/download/win and run the installer.

Key installer options:
- Select "Git from the command line and also from 3rd-party software" (adds Git to PATH)
- Choose "Use Visual Studio Code as Git's default editor" (if you have VS Code)
- Choose "Git from the command line and also from 3rd-party software"
- For "Choosing HTTPS transport backend", select "Use the native Windows Secure Channel library"
- For "Configuring the line ending conversions", select "Checkout Windows-style, commit Unix-style"

**Option B — Winget:**
```
winget install --id Git.Git -e --source winget
```

**Option C — Chocolatey:**
```
choco install git
```

After installation, open a new Command Prompt or PowerShell and verify:
```
git --version
```

### 4. Post-Installation Configuration

Set your name and email (required for commits):
```
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```

Set your default branch name:
```
git config --global init.defaultBranch main
```

Set your preferred editor:
```
git config --global core.editor "code --wait"
```

Verify the configuration:
```
git config --list
```

### 5. Verifying the Tutorial Works

This tutorial uses a sandbox that creates temporary Git repositories. Ensure you can create and commit in a test repo:

```
cd /tmp
mkdir test-install
cd test-install
git init
echo "hello" > README.md
git add README.md
git commit -m "Test commit"
git log
```

If you see a commit entry, Git is correctly installed and configured. This tutorial's sandbox works exactly the same way — every command you type in the integrated terminal panel runs in a real Git repository.

### 6. Troubleshooting

**"git: command not found"** — Git is not installed or not in your PATH. See the installation steps above for your OS.

**"Please tell me who you are"** — Run `git config --global user.name` and `git config --global user.email` as shown in section 4.

**"fatal: not a git repository"** — You need to run `git init` first. The sandbox does this automatically.

**Permission denied (publickey)** — You are trying to push/pull via SSH without configured keys. For this tutorial, focus on local operations (init, add, commit, branch, merge). Remote operations are covered in Phase 3.
